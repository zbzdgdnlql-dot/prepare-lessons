# Français Pro v2.0 技术实现指南 (For Dev AI)

本指南旨在为技术开发侧提供明确的架构调整方向和接口设计规范，以实现 v2.0 产品规划中的 P0 级核心需求。

## 一、 核心需求概述
1. **模块化生成（Checklist）**：前端可选生成哪些模块（如：词汇、语法、习题等）。
2. **卡片式展示与局部重生成**：第三步结果页按模块分块展示，支持针对单一模块重新调用大模型生成。
3. **难度分级与自定义 Prompt**：生成时需带入“CEFR欧标难度（如A1-B2）”和“教师自定义附加指令”。

---

## 二、 后端接口改造方案 (FastAPI)

### 1. 废弃或重构原有的 `generate-lesson` 接口
原接口是一次性生成一段巨大的 Markdown 字符串，这不满足“卡片式”和“局部重生成”的需求。我们需要将其拆解。

### 2. 新增核心生成接口 `/api/generate-module`
为了实现高内聚低耦合，建议**后端提供一个统一的“模块生成接口”**，前端根据用户勾选的模块，**并发（Promise.all）**调用该接口。这样不仅能实现模块化，还能大幅缩短整体等待时间（原本生成整个教案要 30 秒，现在并发生成可能只要 10 秒）。

**请求模型 (Pydantic Model):**
```python
class ModuleGenerateRequest(BaseModel):
    module_type: str        # 必填，例如: "vocabulary", "grammar", "culture", "exercise"
    source_text: str        # 必填，第一步提取的原始文本
    outline: dict           # 必填，第二步确认的大纲 {grammar: [], vocabulary: []}
    cefr_level: str         # 选填，默认 "B1" (如 A1, A2, B1, B2)
    custom_prompt: str      # 选填，教师的附加要求
```

**响应模型:**
```python
class ModuleGenerateResponse(BaseModel):
    module_type: str
    content_md: str         # 该模块生成的具体 Markdown 内容
```

### 3. Prompt 模板的动态拼接逻辑
后端的 `generate_module` 路由中，需要根据 `module_type` 加载不同的 System Prompt。

**伪代码示例：**
```python
# 1. 基础系统指令
system_prompt = f"你是一个专业的法语教育专家。请根据提供的文本和大纲，生成【{module_type}】部分的教学内容。"

# 2. 注入难度约束
if cefr_level:
    system_prompt += f" 你的目标学生水平是 {cefr_level} 级别（CEFR标准）。"
    if module_type == "exercise":
         system_prompt += " 请确保出的习题难度严格符合该级别标准！"

# 3. 注入用户自定义指令
if custom_prompt:
    system_prompt += f" 教师提出了以下特别要求，请务必遵守：{custom_prompt}"

# 4. 规定输出格式
system_prompt += " 请直接输出 Markdown 格式的正式内容，不要包含多余的客套话。"
```

## 三、 前端架构改造方案 (React + Tailwind)

### 1. State 状态管理调整
原来的 `const [lessonPlan, setLessonPlan] = useState<string>('')` 必须废弃，改为字典或数组结构来存储各个模块的状态。

```typescript
// 定义模块的可用类型
type ModuleType = 'vocabulary' | 'grammar' | 'culture' | 'exercise';

interface ModuleState {
    type: ModuleType;
    title: string;
    content: string;     // Markdown 文本
    isLoading: boolean;  // 用于局部重生成时的 Loading 状态
}

// 状态管理
const [modules, setModules] = useState<ModuleState[]>([]);
const [selectedModules, setSelectedModules] = useState<ModuleType[]>(['vocabulary', 'grammar']); // 第二步的勾选状态
const [cefrLevel, setCefrLevel] = useState('B1');
const [customPrompt, setCustomPrompt] = useState('');
```

### 2. 第二步 (UI 新增)
* 增加一个 `<select>` 用于选择 CEFR 级别 (A1, A2, B1, B2)。
* 增加一个 `<textarea>` 用于输入附加指令。
* 增加一组 Checkbox，让用户勾选想要生成的模块。

### 3. 点击“生成”时的并发请求逻辑
在第二步点击“生成”时，前端遍历 `selectedModules`，并发发起请求：

```typescript
const handleGenerate = async () => {
    setStep(3); // 先跳到第三步展示骨架屏
    
    // 初始化 modules 状态，所有被选中的模块状态置为 isLoading: true
    const initialModules = selectedModules.map(type => ({ type, title: getTitleByType(type), content: '', isLoading: true }));
    setModules(initialModules);

    // 并发请求
    await Promise.all(selectedModules.map(async (type) => {
        try {
            const res = await fetch('/api/generate-module', {
                method: 'POST',
                body: JSON.stringify({
                    module_type: type,
                    source_text: inputText,
                    outline: outline,
                    cefr_level: cefrLevel,
                    custom_prompt: customPrompt
                })
            });
            const data = await res.json();
            
            // 请求成功后，更新对应模块的 content，并设置 isLoading: false
            setModules(prev => prev.map(m => m.type === type ? { ...m, content: data.content_md, isLoading: false } : m));
            
        } catch(e) {
            // 错误处理
        }
    }));
};
```

### 4. 第三步卡片化渲染 (UI 改造)
* 遍历 `modules` 数组渲染卡片。
* 如果 `module.isLoading` 为 true，展示一个好看的骨架屏（Skeleton）或 Loading Spinner。
* 如果为 false，使用 `<ReactMarkdown>` 渲染 `module.content`。
* 卡片右上角添加一个“🔄 重新生成”按钮。点击该按钮时，只将该卡片的 `isLoading` 设为 true，并单独发起一次 fetch 请求，成功后替换内容。

## 四、 核心难点提示
1. **大纲结构传递**：在并发调用单个模块时，依然要把完整的 `outline`（包含所有词汇和语法点）传给后端，因为比如“生成习题”时，AI 需要知道本课的核心语法点是什么才能出题。
2. **Token 并发限制**：如果 DeepSeek API 对并发请求有限制（Rate Limit），前端的 `Promise.all` 可能需要改为限制并发数（如 p-limit）或者改为串行生成（一个个出）。建议先用并发测试，如果报错再降级。
