# MVP多语种扩展（一期）：西班牙语与德语技术落地指南

本阶段目标是将纯法语的MVP产品扩展支持**西班牙语（ES）**与**德语（DE）**。
西语与法语在语言体系和UI排版上较为接近，主要作为多语言框架的验证；德语将作为首次“长单词/爆版”压测对象，验证前端UI的弹性适配能力。

---

## 核心挑战与差异点

1. **语言评级标准（统一兼容）**
   - 法、西、德均采用 **CEFR（欧洲语言共同参考框架）**：A1, A2, B1, B2, C1, C2。
   - **优势**：本期底层数据结构（如课程等级、用户水平评估）的排序和权重逻辑暂无需大改，可保持现有的1-6线性递增逻辑。
2. **UI排版与文案长度（德语预警）**
   - **德语（超长单词）**：复合词极长（如 *Benutzerkonto-Einstellungen*），按钮、卡片标题极易出现溢出或换行错位。
   - **西语**：文本长度通常比英语/法语长20%左右。
3. **字符集与变音符号**
   - 西语（á, é, í, ó, ú, ñ, ü）与德语（ä, ö, ü, ß）。需确保字体和数据库字符集完美支持。
4. **复数与动态参数**
   - 这三种语言（法、西、德）的复数形态相对简单（主要区分单数与复数）。

---

## 落地执行步骤 (Step-by-Step)

### 第一步：基础设施建设（代码解耦）
**目标**：剥离代码中硬编码的法语，引入标准的国际化（i18n）框架。

1. **选型与引入**
   - 前端：引入主流的 i18n 库（如 React 的 `react-i18next` 或 Vue 的 `vue-i18n`）。
   - 后端：配置语言中间件，确保接口能够解析 `Accept-Language` 请求头。
2. **提取法语基准字典**
   - 扫描现有代码库，将所有写死的法语文本（包括UI文案、校验提示、邮件模板）抽离为 `fr.json`。
   - 建立键值对命名规范，例如按模块划分：`"login.button.submit": "Se connecter"`。
3. **建立翻译协同机制**
   - 使用在线协同工具（如 Lokalise、Crowdin 或简单的共享表格），将 `fr.json` 作为基准，翻译并生成 `es.json` 和 `de.json`。

### 第二步：业务逻辑与状态改造
**目标**：实现用户语种偏好的记录与切换。

1. **数据库字段扩充**
   - 在用户表（User/Profile）新增 `display_language`（界面语言，枚举值：`fr, es, de`）。
   - 如果产品区分“界面语言”和“学习语言”，需额外增加 `learning_language` 字段。
2. **URL与路由改造（利于SEO）**
   - 采用路径前缀模式：`domain.com/fr/...`、`domain.com/es/...`、`domain.com/de/...`。
   - 配置默认回退逻辑：若未匹配到支持的语言，回退至 `fr` 或浏览器默认语言。
3. **日期、时间与数字格式化**
   - 停用硬编码的格式化函数。
   - 统一采用国际化 API（如原生的 `Intl.DateTimeFormat` 和 `Intl.NumberFormat`）来处理不同语言的千分位（如西语/德语的逗号和小数点习惯与法语有细微差别）。

### 第三步：前端UI弹性适配（针对德语与西语）
**目标**：解决文案变长带来的“爆版”问题，提升多语言界面的健壮性。

1. **破除写死的高宽**
   - 审查所有 CSS，**禁止**在按钮、卡片、导航栏中使用固定的 `width` 和 `height`。
   - 改用 `min-width`、`min-height` 配合 `padding`，让容器随文本自适应。
2. **处理文本溢出与换行**
   - 针对确实受限的空间（如表格列、小卡片标题），增加截断与提示：
     ```css
     .text-truncate {
       white-space: nowrap;
       overflow: hidden;
       text-overflow: ellipsis;
     }
     ```
   - 对于允许换行的长文本，增加：`word-break: break-word; overflow-wrap: anywhere;` 防止长单词冲破容器。
3. **字体回退栈（Font Stack）检查**
   - 确保所选用的 Web 字体包含完整的拉丁字母扩展（Latin Extended），完美显示 `ñ`, `ß` 等字符。

### 第四步：测试与验收
1. **伪本地化压测（Pseudo-localization）**
   - 在翻译包未完成前，前端可利用工具自动将法语文本拉长 40% 并插入特殊符号（如 `[!!! Loooong Texte !!!]`），提前排查UI溢出。
2. **多终端样式走查**
   - 重点检查移动端（H5/App）下，德语长文案在弹窗、底部导航栏（Tab Bar）等狭小空间的表现。
3. **全链路回归**
   - 注册、登录、核心学习流程、支付页面的文案切换是否流畅。
   - 邮件触发器（SendGrid/AWS SES 等）是否根据用户的 `display_language` 发送了对应语种的邮件。
