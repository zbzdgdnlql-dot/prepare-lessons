# Français Pro v2.1 进阶功能改进与技术落地文档

基于最新的产品打磨需求，为了进一步提升教师对备课包生成的**绝对控制权**与**实际授课落地能力**，本阶段将重点攻克以下 4 个核心功能。

---

## 一、 核心改进需求与业务逻辑设计

### 1. 大纲的深度编辑：手动增减语法和单词
* **痛点**：AI 提取的大纲可能有遗漏，或者提取了教师认为不重要的超纲词汇。目前的第二步只允许“确认”，不允许“修改”。
* **产品落地点**：【第二步：大纲确认页】
* **业务逻辑**：
  * **删除**：在每个提取出的单词徽章（Badge）和语法列表项旁边，增加一个 `x` 删除按钮。
  * **新增**：在单词和语法模块的底部，分别增加一个“+ 添加新词汇/语法”的快速输入框。
  * **编辑**：支持双击现有词汇修改其翻译或拼写。
* **价值**：确保传递给第三步生成的“地基”是 100% 准确且符合教师教学意图的。

### 2. 局部重生成的精准调优：手动输入修改意见
* **痛点**：单纯点击“重新生成”就像是在抽盲盒，教师无法告诉 AI “你刚才哪里写得不好”。
* **产品落地点**：【第三步：生成结果页】的卡片操作区。
* **业务逻辑**：
  * 将原来的单一“🔄 重新生成”按钮，改为弹出式交互。
  * 点击“重新生成”时，弹出一个气泡或小弹窗：**“请告诉 AI 您的修改意见（选填）”**。
  * 教师可以输入诸如：“这篇文化背景太长了，缩短到200字以内”或“这几道测验题太简单了，增加几道从句填空题”。
* **价值**：实现真正的“人机协同微调”，大幅提升重生成的满意度。

### 3. 授课物料闭环：落实 Word / PDF（教师版 & 学生版）导出
* **痛点**：Markdown 格式无法直接用于课堂发放，教师极度需要“带着答案讲课，发给学生填空”的纸质/电子档物料。
* **产品落地点**：【第三步：生成结果页】顶部悬浮操作栏。
* **业务逻辑**：
  * 提供一个 **“导出备课包”** 下拉菜单。
  * **选项 A：导出教师版 (Word / PDF)** —— 包含所有完整内容、答案、精讲解析。
  * **选项 B：导出学生版 (Word / PDF)** —— **核心魔法**：后端需要对 Markdown 文本进行处理，将例句中的关键单词、习题的答案部分替换为下划线 `________`，并去除“教学建议”等纯教师向的内容。
* **技术挑战**：前端渲染 Markdown 转 Word/PDF 的保真度；“挖空”算法的准确性。

### 4. 极致透明化：后端 Prompt 提示词的透出与手动微调
* **痛点**：黑盒式的 AI 提示词让资深教师无法发挥自己的教育经验。他们需要看到系统是怎么向 AI 下指令的，并在必要时干预。
* **产品落地点**：【第二步与第三步生成前】或【全局设置页】。
* **业务逻辑**：
  * 在页面顶部提供一个 **“开发者/高级设置：查看与修改模型提示词”** 的开关。
  * 打开后，在触发生成请求前，弹出一个面板，展示即将发送给 DeepSeek 的完整 System Prompt。
  * 教师可以直接在这个文本框里修改 Prompt 模板（例如修改角色定义、修改输出的语气要求），然后再点击“确定生成”。
* **价值**：赋予高级用户最高级别的控制权，同时也是极好的“Prompt Engineering”教育过程。

## 二、 技术实现指南 (For Dev AI)

### 1. 大纲增减功能实现 (React 前端)
* **状态管理**：`outline` 状态不再是只读的。
  ```typescript
  // 增加删除逻辑示例
  const removeVocabulary = (index: number) => {
    setOutline(prev => ({
      ...prev,
      vocabulary: prev.vocabulary.filter((_, i) => i !== index)
    }));
  };
  ```
* **UI 组件**：需引入受控的 Input 组件用于新增词汇/语法，并维护临时状态，按回车键或点击“添加”后推入 `outline` 数组。

### 2. 重生成带附加意见实现 (前后端联调)
* **前端**：在调用 `/api/generate-module` 接口时，新增一个字段 `feedback: string`。
* **后端**：
  ```python
  if request.feedback:
      system_prompt += f"\n\n用户对上一次的生成结果不满意，提出了以下修改意见，请务必严格遵照执行：\n{request.feedback}"
  ```

### 3. Word / PDF 双版本导出落地实现
建议将繁重的文档排版与“挖空”工作放在**后端**处理，因为 Python 有非常成熟的文档处理库。
* **新增接口**：`/api/export-document`
* **Python 依赖**：
  * Word 导出：`python-docx` + `pypandoc` (用于将 Markdown 转为 Docx 格式)。
  * PDF 导出：可以使用 `pdfkit` (基于 pandoc) 或先生成 Word 再转 PDF。
* **学生版“挖空”算法 (NLP 简易版)**：
  * **规则匹配**：如果 Markdown 中有 `**加粗的词**`（通常是重点词汇或答案），在生成学生版时，用正则将其替换为 `______`。
  * **AI 挖空**：或者再调用一次极其便宜的模型 API（如 DeepSeek-Flash），指令为：“将以下教学讲义转为学生填空版，隐藏答案和关键动词”。

### 4. 提示词透出与微调实现 (架构大改)
这是一个架构上的范式转换：**把 Prompt 的拼接权从后端部分转移到前端/用户手中。**
* **重构思路**：
  1. 后端提供一个接口 `/api/get-prompt-template`，返回各模块的默认 Prompt 模板。
  2. 前端获取并在“高级设置”中展示这些模板。
  3. 教师修改后，前端将其保存在 State 或 LocalStorage 中。
  4. 发起生成请求时，前端将**最终拼接好的完整 System Prompt**直接通过接口发给后端，后端只负责透传给大模型。
* **接口变更**：
  ```python
  class ModuleGenerateRequest(BaseModel):
      # ... 其他参数 ...
      # 新增参数，如果前端传了，后端就直接用这个作为 System Prompt，不再自己拼接
      override_system_prompt: Optional[str] = None 
  ```
