# Français Pro v2.2 改进与功能实现记录

## 1. 修复 Markdown 格式单词显示问题
- **问题描述**：在生成最终教案环节，部分单词或语法表格（Markdown格式）未能正确渲染，影响阅读体验。
- **技术实现**：
  - 在前端引入了 `remark-gfm` 插件（GitHub Flavored Markdown）。
  - 在 `App.tsx` 中的 `<ReactMarkdown>` 组件里配置 `remarkPlugins={[remarkGfm]}`，使系统原生支持 Markdown 表格、删除线等复杂格式的正确渲染，确保生成的词汇和语法表格排版整洁。

## 2. 优化 Word 文档导出排版效果
- **问题描述**：此前直接将 Markdown 写入 `.docx` 导致最终生成的 Word 文档缺乏格式，排版混乱。
- **技术实现**：
  - 在后端引入了 `pypandoc` 库（基于强大的文档转换工具 Pandoc）。
  - 修改 `main.py` 的 `/api/export-document` 接口，先将内容保存为临时 `.md` 文件，然后调用 `pypandoc.convert_file` 将其标准、高质量地转换为带格式的 `.docx` 文件。
  - 此改进使导出的 Word 文档完整保留了标题层级、加粗、表格和列表等富文本排版。

## 3. 生成习题模块增加子题型配置（专四标准）
- **需求描述**：习题模块过于宽泛，希望能按专四题型标准选择性生成特定习题。
- **技术实现**：
  - **前端 (`App.tsx`)**：
    - 新增 `exerciseTypes` 状态，内置听写、听力理解、词汇与语法、完形填空、阅读理解、写作等专四标准题型供用户多选。
    - 在“生成选项”中，若勾选了“随堂测试与习题”，则动态展开具体的题型勾选面板。
    - 将用户选中的题型作为参数 `exercise_types` 传给后端。
  - **后端 (`main.py`)**：
    - 扩展 `ModuleGenerateRequest` 数据模型，新增 `exercise_types` 字段。
    - 在生成 prompt 时，动态读取所选题型并写入指令（例如：“必须且仅包含以下题型（专四标准）：阅读理解、完形填空”）。

## 4. 新增模块：课堂活动
- **需求描述**：增加贴合文章内容的课堂互动活动，且教师可以做自定义要求。
- **技术实现**：
  - **前后端协同**：
    - 在前端 `ModuleType` 和后端的 `module_names` 中注册新的模块类型：`activity` (课堂互动活动)。
    - 前端生成选项中添加该模块供用户勾选。
  - **Prompt 设计 (`main.py`)**：
    - 针对该模块设计了专属系统提示词：“设计与文本内容和语言点高度贴合的【课堂互动活动】。要求提供至少两个不同的活动（如角色扮演、辩论等），明确列出活动目的、所需时间、操作步骤和教师引导语，并确保符合目标学生的 CEFR 级别。”
  - 同样支持教师在前端使用“自定义额外要求”框进行细节微调（如限定活动时间为10分钟）。

## 5. 教师收藏功能
- **需求描述**：让教师可以收藏生成的某些不错的讲解和习题。
- **技术实现**：
  - **前端 (`App.tsx`)**：
    - 在生成的每一个模块操作区（“重新生成”旁）增加一个“收藏”按钮。
    - 使用浏览器的 `localStorage` (`fp_bookmarks`) 持久化存储用户的收藏内容，无需额外搭建数据库即可实现跨会话的收藏保留。
    - 在顶部导航栏新增“我的收藏”入口，点击弹出收藏库 Modal（弹窗），支持按时间线查看所有收藏的模块。
    - 收藏库内支持“一键复制内容”和“删除收藏”操作，极大方便了教师将优质片段拼接到自己的教案中。
